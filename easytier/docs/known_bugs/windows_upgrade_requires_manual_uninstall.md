# Windows 原地升级需要手动卸载旧版本

## 状态

**已在 v2.6.9 安装包升级场景观察到；本轮不修复。**

## 现象

Windows 上直接运行新版安装包覆盖旧版时，安装可能失败或无法继续。当前可复现的
规避流程是：

1. 退出 EasyTier GUI，并停止正在运行的网络实例。
2. 在 Windows“服务”中停止 EasyTier 对应服务。
3. 卸载旧版 EasyTier。
4. 确认 EasyTier GUI、Core 和服务进程均已退出。
5. 安装新版本。

执行卸载前应备份重要配置。不要为了完成升级而手工删除未知的配置、驱动或系统文件。

## 影响

- 当前不能保证 Windows 安装包支持无人工干预的原地升级。
- 旧服务或进程仍持有程序文件、驱动或服务注册状态时，新安装可能被阻塞。
- 该问题影响安装/升级流程，不代表已安装版本的数据面或网络协议存在故障。

## 尚未确认

当前证据不足以确定问题位于安装器进程终止、Windows 服务停止等待、文件占用、驱动
卸载，还是升级标识/安装器元数据。修复前必须分别采集安装器日志、服务状态、残留进程、
文件锁和安装包 upgrade code，不能仅通过强制终止服务掩盖根因。

## 后续验收标准

- 旧版 GUI、Core 和服务运行时启动新版安装包，安装器能够提示并安全停止相关组件。
- 无需手动卸载即可完成升级，配置得到保留。
- 升级后只有一个服务注册和一组有效进程，不残留旧二进制、驱动、临时目录或启动项。
- 覆盖 Windows x86、x64、ARM64，以及“服务未运行、服务运行、GUI 运行、异常退出后
  残留”场景。
