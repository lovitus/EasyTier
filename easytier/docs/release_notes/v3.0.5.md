# EasyTier v3.0.5 Release Notes

EasyTier 3.0.5 fixes lifecycle failures in Android VPN ownership and Leaf mesh
policy actors, and makes the formal Linux/macOS packages self-contained for the
managed Leaf/HEV policy path.

## Changes

- When Android revokes EasyTier's VPN ownership because another VPN takes over,
  EasyTier now stops and disables the owning non-`no_tun` network instance. This
  also tears down its policy runtime instead of leaving the network shown as
  running with stale latency updates. If the WebView misses the native callback,
  the persisted revoke marker performs the same cleanup when synchronization
  resumes.
- A Leaf mesh policy actor whose selected EasyTier node is not currently present
  no longer prevents the complete Leaf runtime from starting. The unavailable
  actor remains isolated; only traffic selecting it fails until a matching mesh
  route becomes available, while other policy actors and rules continue to run.
- Formal Linux x86_64/aarch64 and macOS x86_64/aarch64 Core packages now include
  the policy feature, Leaf worker and pinned managed HEV egress. Linux and macOS
  GUI packages bundle the same sidecars instead of relying on an external
  executable. The Linux aarch64 Magisk module includes them as well.
- Managed HEV uses the configured Linux socket mark and the configured macOS
  outbound interface. Remote prepare failures are surfaced; fixed-port fallback
  remains only for mixed deployments whose older peer lacks the prepare RPC.

## 中文摘要

- Android 上其他 VPN 抢占系统 VPN 所有权后，EasyTier 会停止并禁用对应的
  非 `no_tun` 网络实例，同时清理 policy runtime，不再保留虚假的运行状态和
  延迟刷新。WebView 暂停而错过回调时，也会在恢复同步后根据持久化撤销标记
  完成相同清理。
- Leaf 配置引用的 mesh 节点暂时不存在时，不再导致整个 Leaf runtime 启动
  失败。只有命中该 actor 的流量失败，其他规则和 actor 继续正常运行；节点
  路由恢复后可重新获得目标。
- 正式 Linux x86_64/aarch64 和 macOS x86_64/aarch64 Core/GUI 制品现在会
  携带 policy 功能、Leaf worker 与固定版本的 HEV egress，不再依赖系统中
  另行安装的可执行文件；Linux aarch64 Magisk 模块也包含相同 sidecar。
- HEV 会继承 Linux policy socket mark 或 macOS 指定出口接口；远端 prepare
  的真实失败会直接报告，只有旧节点不支持该 RPC 时才回退固定端口。
