# EasyTier v3.0.1 Release Notes

EasyTier 3.0.1 fixes policy fallback health management so destination-specific
failures cannot reroute later traffic or temporarily block the whole group.

## External Changes

- A fallback group is now a sticky selector, not a per-request retry list. Every
  TCP or UDP request uses only the currently selected member; failure is returned
  to that request without trying another member.
- Health is determined only through an independent HTTP(S) URL. User connection
  success or failure never changes proxy health.
- Checks are demand-driven. An unused group produces no health traffic, and an
  active group checks only when its last result is at least five minutes old.
- The default URL is `https://www.gstatic.com/generate_204`. A fallback group may
  override it with `url`; the policy editor exposes the same field.
- Checks use `HEAD`, do not follow redirects, and accept status codes below 400.
- Switching and recovery are both stable: the same first-healthy member must win
  three rounds before selection changes. If all members fail, the current member
  remains selected instead of moving traffic unpredictably.

## Implementation Details

- TCP and UDP handlers share one atomic selector and never write health state from
  the user data path.
- Real traffic touching a stale group starts one asynchronous singleflight session;
  the triggering request does not wait. Checks use at most ten concurrent actor
  tasks while preserving strict configured order.
- Confirmation rounds run immediately, after approximately five seconds, and
  after approximately ten more seconds. The rule applies equally to failover and
  failback.
- Inconclusive all-member failures retain the selected actor and use traffic-driven
  suppression of 15 seconds, 60 seconds, then five minutes. No background outage
  timer remains.
- Internal probe context is propagated through chain and nested-group actors so a
  health request cannot recursively trigger another fallback health session.
- The pinned Leaf revision is
  `013a1497dd29355a00cd776628ff2de72e02e861`.

## Compatibility and Upgrade Notes

- Existing fallback YAML remains valid. The optional `url` field is accepted only
  on fallback groups and must be an HTTP(S) URL with a host and no credentials.
- There is intentionally no `expected-status` field. Responses in the 1xx, 2xx or
  3xx range are healthy; 4xx, 5xx and network, TLS or HTTP parsing errors fail.
- A user request that cannot use the selected member now fails directly. This is
  intentional: trying later members could leak an internal destination through an
  unrelated exit and incorrectly treat that exit as the right route.
- Non-stable Leaf failover and policy configurations without fallback groups are
  unchanged.

## Validation

- Locked Rust no-run and focused selector, stream, datagram, configuration and
  fixture tests passed on the remote builder.
- Frontend policy tests passed 16/16 and the production build completed.
- The exact implementation artifact passed Linux and Android candidate workflows.
- lv1g2 and lv1g3 independently passed demand triggering, concurrent singleflight,
  no request-level retry, three-round switch, real five-minute freshness,
  all-fail hold, 15/60-second suppression, three-round failback, nested chain,
  resource and cleanup checks.

## 中文摘要

EasyTier `v3.0.1` 修复策略代理 fallback 的健康管理和切换语义。

- fallback 现在是粘性选择器，不是单次请求的逐节点重试列表。每个 TCP/UDP 请求只走
  当前成员，失败直接返回，不会因为目标地址不可达而切换出口。
- 健康状态只来自独立 HTTP(S) URL；用户业务连接的成功或失败不参与健康判断。
- 没有启动探测或永久定时器。只有真实流量进入已过期的组时，才异步触发一次检查；
  五分钟内继续使用当前选择。
- 默认检查地址为 `https://www.gstatic.com/generate_204`，fallback 可通过 `url`
  覆盖。使用 `HEAD`，不跟随重定向，HTTP 状态小于 400 即为健康。
- 连续三轮得到同一个首个健康成员后才切换；恢复更靠前成员时也按三轮确认回切。
- 全组失败保持当前成员，并按 15 秒、60 秒、5 分钟抑制后续检查，避免抖动和持续探测。
