# EasyTier v3.0.0 Release Notes

This release packages the current EasyTier 3.0 policy-proxy and networking
implementation after its locked-builder and multi-host validation gates.

## Highlights

- Adds the default-off Linux Leaf-owned policy TUN fast path, with the original
  EasyTier TUN retained as a fail-closed fallback.
- Includes the expanded Leaf policy stack for Shadowsocks/UoT, Trojan, VMess,
  VLESS, DNS/FakeDNS handling, mesh proxy composition, and strict rule parsing.
- Normalizes Mihomo-compatible single-port `PORT-RANGE` rules to Leaf's explicit
  `start-end` representation and rejects invalid or descending ranges.
- Keeps unsupported platforms and modes on the existing legacy policy data path.
- Adds profiling-only isolated Linux self-tests for Leaf DIRECT and native, KCP,
  QUIC, and forced-relay mesh paths. These tools are excluded from production
  release archives and application dependency graphs.

## Validation Boundary

The exact release candidate is required to pass the locked remote-builder
preflight, profiling artifact integrity checks, and independent Linux functional,
lifecycle, resource, cleanup, and cross-host smoke tests. The Leaf-owned TUN
optimization remains explicit and default-off.

## 中文摘要

EasyTier `v3.0.0` 包含完成锁定构建器和多主机验证门禁后的 3.0 policy-proxy 与网络实现。

- 提供默认关闭的 Linux Leaf-owned policy TUN 快速路径，并保留原 EasyTier TUN
  作为 fail-closed 回退。
- 包含 Shadowsocks/UoT、Trojan、VMess、VLESS、DNS/FakeDNS、mesh proxy 组合和
  严格规则解析等 Leaf policy 功能。
- 单端口 `PORT-RANGE` 会规范化为 Leaf 所需的显式范围，无效或倒序范围会被拒绝。
- 未支持的平台和运行模式继续使用原 legacy policy 数据路径。
- profiling 制品包含隔离的 Linux 自检工具；这些工具不会进入正式发布包或应用依赖图。
