# EasyTier v3.0.0 Release Notes

EasyTier 3.0 introduces an optional, first-match policy proxy while preserving
the existing mesh data plane for configurations that do not enable it.

## Highlights

- Adds Leaf policy routing with DIRECT, REJECT, SOCKS5, Shadowsocks/UoT,
  Trojan, VMess and VLESS outbounds, including WebSocket/TLS transport where
  supported by the pinned Leaf implementation.
- Supports native and mesh-composed proxy hops, chains and fallback groups.
  KCP and QUIC remain mesh-owned transport choices rather than policy-owned
  protocol semantics.
- Adds domain, IP-CIDR, PORT-RANGE, GeoSite, GeoIP and first-match fallback
  rules. Single-port `PORT-RANGE` input is normalized to an explicit range;
  invalid and descending ranges are rejected.
- Adds policy DNS/FakeIP configuration, separate direct/proxy resolver presets,
  configurable IPv4/IPv6 FakeIP pools and bounded mapping limits.
- Adds cached GeoSite/GeoIP category discovery through core RPC, Web REST and
  Tauri. The index is reused while rule-data identity and size are unchanged and
  is rebuilt after source changes or cache corruption.
- Makes policy nodes, groups and rules compact by default in the GUI, adds field
  examples, and corrects protocol and UDP capability labels.
- Adds a default-off Linux Leaf-owned policy TUN fast path. Unsupported modes or
  failed initialization retain the existing legacy policy path; the normal
  EasyTier mesh TUN, KCP and QUIC behavior is not replaced.
- Adds profiling-only isolated Linux probes for DIRECT and native, KCP, QUIC and
  forced-relay mesh paths. These tools are not part of production application
  dependency graphs.

## Compatibility and boundaries

- Existing configurations without policy proxy fields continue to use their
  defaults; default DNS presets are supplied when older policy YAML omits `dns`.
- Policy proxy remains optional. Enabling it adds a second routing/runtime layer;
  users should retain a known-good configuration when upgrading.
- Protocol fields are validated against the subset implemented by EasyTier and
  its pinned Leaf revision. Unsupported combinations fail during configuration
  validation rather than being silently accepted.
- The Linux Leaf-owned TUN optimization is experimental and default-off.
- Android physical-device qualification is a release gate. This release must not
  be published until the exact candidate is either tested successfully or the
  maintainer explicitly records a waiver in the validation matrix. A Mobile
  compile alone is not runtime evidence.

## Upgrade guidance

Before enabling policy routing broadly, test the intended DNS, chain/fallback,
Geo and IPv4/IPv6 behavior on a non-critical instance. If policy startup fails,
disable the policy feature and keep the existing mesh configuration unchanged.

## 中文摘要

EasyTier `v3.0.0` 新增可选的 Leaf 策略代理，同时保持未启用策略代理时的原有
mesh 数据面和配置兼容性。

- 支持 DIRECT、REJECT、SOCKS5、Shadowsocks/UoT、Trojan、VMess、VLESS，
  以及 native/mesh 组合、chain 和 fallback。
- 支持域名、IP-CIDR、PORT-RANGE、GeoSite、GeoIP 和首条匹配规则。
- 提供 direct/proxy DNS、可配置 IPv4/IPv6 FakeIP 地址池和有界映射数量。
- GUI 默认折叠节点、组和规则，补充字段示例、协议说明及 Geo 分类缓存索引。
- Linux Leaf-owned policy TUN 快速路径默认关闭；不支持或初始化失败时保留原有
  legacy 路径，不接管 EasyTier mesh 的 KCP/QUIC 选择。
- Android 实机是发布门禁；必须对精确候选通过验证，或由维护者明确记录豁免，
  不能用 Mobile 编译结果替代实机证据。
