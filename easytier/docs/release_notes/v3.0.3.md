# EasyTier v3.0.3 Release Notes

EasyTier 3.0.3 makes the experimental `quic-brutal` transmit-rate setting
human-readable without changing its transport, Stealth, or fallback behavior.

## Changes

- GUI, CLI, TOML and RPC URLs now use `tx_mbps` in Mbps, for example
  `quic-brutal://SERVER:11013?tx_mbps=100`.
- The valid range is 1 through 100,000 Mbps. Up to six decimal places are
  accepted when an exact non-integer Mbps value is needed.
- Existing 3.0.2 URLs containing `tx_bps` remain accepted as a compatibility
  alias. New and edited GUI configurations are saved with `tx_mbps`.
- Supplying both parameters, duplicating either parameter, using an unsupported
  parameter, or specifying an out-of-range value fails explicitly.
- Omitting the transmit rate continues to select ordinary QUIC BBR for that
  local sending direction.

The setting remains directional: a listener value controls traffic sent by the
listening node, while a peer URL value controls traffic sent by the dialing
node. `quic-brutal` remains opt-in and ordinary `quic://` behavior is unchanged.

## 中文摘要

- GUI、命令行、TOML 和 RPC URL 统一使用 Mbps 单位的 `tx_mbps`，例如
  `quic-brutal://SERVER:11013?tx_mbps=100`。
- 合法范围为 1 到 100,000 Mbps，最多支持 6 位小数。
- 3.0.2 的 `tx_bps` 配置继续兼容读取；新建或经 GUI 编辑的配置保存为
  `tx_mbps`。
- 两个参数混用、参数重复、参数不支持或数值越界都会明确报错。
- 留空仍让本节点发送方向使用普通 QUIC BBR；`quic-brutal` 仍需显式启用，
  普通 `quic://` 行为不变。
