# EasyTier v3.0.2 Release Notes

EasyTier 3.0.2 adds an experimental, explicitly enabled `quic-brutal` mesh
overlay while preserving the existing QUIC transport and its BBR behavior.

## External Changes

- Adds the private EasyTier-only `quic-brutal://` overlay. It is not compatible
  with standard Hysteria2 clients or servers.
- The overlay is opt-in and is not added to the default CLI listener set or the
  existing transport priority order.
- The GUI can add a `quic-brutal` listener using the preconfigured port `11013`.
- When QUIC Stealth is configured, `quic-brutal` uses the same authenticated
  outer UDP wrapper and the same required/legacy-fallback negotiation as ordinary
  QUIC; Brutal pacing and Stealth remain independently selected.
- An optional `tx_bps` URL parameter selects Brutal pacing for the local sending
  direction. The value is in bit/s and must be between `1,000,000` and
  `100,000,000,000`.
- Omitting `tx_bps` safely uses the same BBR congestion control as ordinary QUIC
  for that sending direction. EasyTier does not silently guess bandwidth.
- Upgrades `quinn-udp` from `0.5.4` to `0.5.15` and fixes Stealth QUIC handling
  of the final short UDP GSO segment.

## Compatibility and Upgrade Notes

- Existing `quic://` listeners, peers, connection configuration and congestion
  control are unchanged.
- `quic-brutal` uses the existing listener, connector, transport selection and
  failure handling paths. It adds no protocol-specific fallback policy.
- A required Stealth connection never falls back to plain Brutal. Where the
  existing unknown-capability policy allows legacy fallback, a failed Brutal
  Stealth attempt retries plain Brutal without changing its congestion control.
- `tx_bps` is local sender state. A listener value controls traffic sent by the
  listening node; a peer URL value controls traffic sent by the dialing node.
- The feature is experimental. Validation found improvements on some paths but
  not a universal bidirectional gain, so it is not enabled automatically.

## Basic Usage

```bash
# Listening node: allow this node to send at a configured 1 Gbit/s rate.
easytier-core --listeners 'quic-brutal://0.0.0.0:11013?tx_bps=1000000000'

# Dialing node: configure its local sending rate as 100 Mbit/s.
easytier-core --peers 'quic-brutal://SERVER:11013?tx_bps=100000000'

# Confirm the active tunnel type in the tunnel column.
easytier-cli peer list
```

## 中文摘要

- 新增 EasyTier 私有、实验性的 `quic-brutal://` mesh overlay，不兼容标准
  Hysteria2 客户端或服务端。
- 该协议不会默认启用，也不会改变普通 `quic://` 的 BBR、配置或优先级。
- GUI 可显式选择 `quic-brutal`，预配置端口为 `11013`。
- 启用 QUIC Stealth 时，`quic-brutal` 复用相同的外层认证封装和现有协商语义；
  强制 Stealth 不会降级，只有现有未知能力兼容模式才允许回退到 plain Brutal。
- `tx_bps` 单位为 bit/s，只控制本节点发送方向；留空时该方向安全回退到 BBR，
  不会自动估算带宽。
- 可通过 `easytier-cli peer list` 的 `tunnel` 列确认连接是否实际使用
  `quic-brutal`。
