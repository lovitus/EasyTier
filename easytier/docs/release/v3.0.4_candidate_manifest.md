## Intended build snapshot

- Adds a home-page policy-routing toggle and focused YAML editor. Saving changes
  configuration only and explicitly requires a network restart before it takes
  effect.
- Reports the independently owned Leaf policy runtime as running or stopped via
  `NicCtx::policy_runtime_running`, `InstanceRpcService`, protobuf field 11 and
  the existing conservative status refresh path.
- Adds a floating start/stop action that reuses the existing lifecycle methods;
  the original page actions remain available.
- Restores the system context menu in production WebView so Android text fields
  can expose copy, paste and selection actions.
- Lets GeoSite, GeoIP and MMDB sources be edited and updated manually, displays
  their saved SHA-256, and skips a same-size download only when a valid saved
  category index from a previously successful validation is available.
- Keeps failed, truncated, oversized or unparseable rule-data downloads away
  from the active file and does not save a hash/index until validation succeeds.
- Includes focused Rust and frontend regression tests, mobile header wrapping,
  `3.0.4` version metadata, release notes and release-audit pins. It does not
  change Leaf/HEV dependency pins or any existing overlay transport.

The unrelated local `upstream_7_commits_selective_integration_plan.md` remains
outside this release candidate.
