## Superseded exact-artifact evidence

The following evidence remains valid for code behavior but cannot qualify the
final release after a tracked harness change:

- Exact bridge SHA `f76ff6ff4d21a9561cd1cd378893bbcd7bac8372`, Linux run
  `29711421397`, Android run `29711421398`, comparator run `29711437998`.
- `.37` and `.38`: exact 16 MiB DIRECT, native, KCP, QUIC and forced-relay
  matrices passed with bounded resources and complete cleanup.
- lv1g2 and lv1g3: exact 128 MiB isolated matrices passed. The sole original
  lv1g3 host-state failure was a six-second DHCP lifetime countdown; the corrected
  harness passed against the unchanged product artifact.
- Cross-host 10 Gbps physical IPv4/IPv6 baselines exceeded 7.2 Gbit/s. EasyTier
  IPv4-underlay and IPv6-underlay runs both carried dual-stack overlays; their
  medians ranged from 182 to 395 Mbit/s depending on family and direction.
- Disabled-mode A/B: three feature-on and three no-Leaf runs all passed. Upload
  medians were about 1.11 and 1.07 Gbit/s; download medians were about 1.13 and
  1.09 Gbit/s respectively. Per-process RSS remained 22-26 MiB, FD 24 and
  threads 9-10, with overlapping CPU ranges.

## Android waiver contract

The maintainer permits a waiver mechanism, not an automatic waiver. The exact
candidate SHA, unavailable device reason, retained build evidence and missing
runtime scenarios must be recorded here and disclosed in the final release notes
before the state may become `WAIVED_BY_MAINTAINER`.
