## Why the previous candidate is invalid

The profiling candidate contained the Leaf feature, Leaf worker and HEV egress,
but the formal Core and GUI workflows did not build the same policy-capable
payload. Formal Core omitted `leaf-policy-proxy` and both sidecars. macOS GUI
bundled only Leaf, Linux GUI enabled neither the policy feature nor the
sidecars. Those workflows could succeed while the installed product could not
start managed HEV. Their successful run results remain diagnostic evidence but
must not be used to authorize a tag or release.

The later frozen candidate `f45fad64` passed its exact Linux and Android
candidate validation, but the formal release gate exposed two independent,
real build failures:

- Windows GUI compilation reached the new policy monitor and failed because
  `std::time::Duration` was imported only for Unix.
- Test/Clippy reached the Tauri GUI build script without the declared
  target-suffixed Leaf/HEV external binaries. Formal GUI builds create and
  inspect the real sidecars, but the static all-workspace check had no explicit
  fixtures for Tauri's existence validation.

The GUI failure cancelled the other GUI matrix entries. Core, Mobile, and Test
were deliberately cancelled after the release gate was known to be invalid;
OHOS succeeded. No tag or GitHub Release was created. None of those formal
results authorizes release of either `f45fad64` or its descendants.

The replacement `3f3c93e3` then passed its exact profiling workflow, Android
workflow, Linux deployment and Android physical-device validation. Formal GUI,
Mobile, OHOS and Test also passed. Formal Core run `30016957223` compiled the
Windows x86_64 Core and Leaf worker successfully, then its newly added runtime
smoke returned exit code `127`. That smoke ran before the architecture-matched
DLLs and driver payload were staged for the final archive, and Git Bash did not
retain a Windows loader diagnostic. Matrix fail-fast then cancelled the
remaining unfinished Core targets. No tag or GitHub Release was created.

The post-`3f3c93e3` repair changes no Rust, dependency, feature or packaged
payload. `.github/workflows/core.yml` now stages the same target-specific
Windows runtime files beside `easytier-core.exe` before executing the smoke and
runs the check through native PowerShell. The gate still requires a zero exit
code, `--policy-config`, the Leaf worker, Wintun and Packet runtime payloads;
PowerShell now preserves the executable output and exact exit code if the
failure is not the staging-order mismatch. Final archive collection remains
unchanged.

## Replacement build snapshot

- Retain the already validated Android VPN-ownership cleanup and missing
  mesh-actor availability behavior from `046ce631`.
- Build Linux x86_64/aarch64 and macOS x86_64/aarch64 formal Core with
  `leaf-policy-proxy`, `easytier-leaf-worker` and
  `easytier-hev-socks-egress`. The aarch64 Magisk module copies the same two
  adjacent sidecars and refuses an incomplete manual release payload.
- Bundle the same two sidecars in Linux and macOS GUI packages. Keep the
  policy/sidecar boundary disabled on Windows and on formal Linux architectures
  for which pinned HEV is not built by this release.
- Replace the unmanaged upstream HEV executable with a thin EasyTier-owned
  wrapper linked to the exact pinned HEV libraries. The wrapper contains no
  SOCKS implementation; it only owns the stable process entry point, embedded
  revision reporting and macOS parent-death watchdog.
- Require adjacent sidecar discovery. Do not search `PATH`, so an unrelated HEV
  binary cannot silently satisfy an incomplete installation.
- Propagate the configured Linux policy socket mark into HEV. On macOS,
  propagate the configured outbound interface so split-default capture does not
  loop HEV's own destination sockets.
- Fall back to legacy fixed port `11080` only when the remote peer lacks the
  prepare RPC. A remote execution error or timeout is now reported instead of
  being mislabeled as an old-peer capability fallback.
- Ad-hoc sign the complete macOS app and raw Core executables, then verify every
  sidecar and the app bundle. Inspect formal Core/GUI payloads before release.
- Treat both `statically linked` and `static-pie linked` as valid musl HEV
  payloads in the build script and profiling workflow, while still requiring
  the embedded revision and bounded execution smoke.
- Include the two safe pre-existing TODO documents in the workspace commit, as
  required by the repository submission rule. They do not change the build.
- Enable the Leaf runtime on Linux, macOS and Windows GUI/Core delivery paths
  without changing the Android in-process boundary. macOS passes IP packet
  information through the external TUN fd, installs interface-scoped capture
  routes and waits for writability at both Darwin `ENOBUFS` boundaries.
  Windows resolves the selected friendly interface alias to its stable LUID and
  current interface index before applying IPv4/IPv6 socket binding.
- Keep the final EasyTier-to-Leaf datagram bridge bounded. Backpressure waits
  for actual sender capacity rather than increasing a fixed queue or retrying
  on a timer.

## Reference behavior and intentional boundaries

- Pinned HEV `97e74f1068bd924e740032382cdc94ca83741ae6`:
  `src/misc/hev-misc.c::set_sock_bind` applies Linux `SO_BINDTODEVICE` and Apple
  `IPV6_BOUND_IF`; `set_sock_mark` applies Linux `SO_MARK`;
  `src/hev-main.c::{hev_socks5_server_main_from_file,
  hev_socks5_server_main_from_str,hev_socks5_server_quit}` supplies the process
  and in-process lifecycle entry points.
- Mihomo `mihomo-rev/component/dialer/bind_darwin.go`
  binds Apple outbound sockets with `IP_BOUND_IF`/`IPV6_BOUND_IF`.
  `mihomo-rev/component/tsnet/tsnet.go::{startSocks5,
  retryStartSocks5TCP,retryStartSocks5UDP,serveSocks5TCP}` owns its SOCKS
  listeners inside the userspace tailnet and retries explicitly. EasyTier's
  corresponding virtual ingress is
  `MeshSocksRelayService::start_local_tcp_ingress`; the separate HEV hop is
  therefore loopback-only.
- Mihomo `mihomo-rev/adapter/outbound/wireguard.go::{updateServerAddr,
  genIpcConf}` refreshes resolved peer endpoints in place and keeps unchanged
  generations alive. EasyTier applies the equivalent behavior at
  `MeshProxyBridgeSet::replace_remote_snapshot`; a missing actor is unavailable
  without changing first-match policy compilation or other actors.
- sing-box `singbox-withfallback/common/dialer/default.go`
  treats outbound interface binding as a dialer option on Linux, macOS and
  Windows.
- EasyTier intentionally keeps HEV as a supervised adjacent process on desktop
  and the same pinned library in-process on Android. This preserves the current
  failure isolation and mobile host boundary without duplicating SOCKS protocol
  logic.
