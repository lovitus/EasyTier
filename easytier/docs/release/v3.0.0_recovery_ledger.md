## Bridge and post-bridge evidence

- Recovery commit `c74c5af98a54ca81d50b5da41254779315b1f808` and public bridge
  commit `f76ff6ff4d21a9561cd1cd378893bbcd7bac8372` have the identical tree
  `bd870f5f0162d6df84ff920dbae9a3b566ef2f7b` with 933 tracked files.
- The bridge contains six independent revert commits followed by the 16 accepted
  recovery commits. It was pushed once to `codex/profiling-beta` without reset or
  force-push.
- Automatic Linux run `29711421397`, Android build run `29711421398`, and the
  comparator-only run `29711437998` all succeeded for exact SHA `f76ff6ff`.
- Exact Linux artifacts passed isolated DIRECT, native, KCP, QUIC and forced-relay
  checks on `.37`, `.38`, lv1g2 and lv1g3. Cross-host IPv4 and IPv6 underlays both
  carried IPv4 and IPv6 overlays without an address-family-specific failure.
- Three feature-on and three no-Leaf native runs used the same 64 MiB harness and
  passed byte, route, transport, resource, amplification and cleanup gates. Their
  throughput, CPU, RSS, FD and thread ranges overlap; disabled mode has no measured
  regression attributable to compiled Leaf support.
- The lease-lifetime normalization is a profiling-harness correction. Because it
  changes a tracked release artifact, it must travel through a new `.160` gate and
  exact-SHA candidate workflow rather than silently inheriting `f76ff6ff` evidence.
