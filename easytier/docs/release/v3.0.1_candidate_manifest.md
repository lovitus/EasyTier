## Included behavior

- Replaces policy fallback's request-failure/outage state machine with a TCP/UDP-shared selector.
- Sends each user request only to the selected member; user destination failures never retry another member or alter group health.
- Runs no startup or periodic checks. Real group traffic triggers one asynchronous check session only when health is stale.
- Uses Mihomo's default `https://www.gstatic.com/generate_204`, supports a fallback-only `url` override, sends `HEAD`, does not follow redirects, and treats HTTP status below 400 as healthy.
- Selects the first healthy member in configured order with at most ten concurrent checks. A different selection requires three identical rounds separated by five and ten seconds.
- Applies the same confirmation to failover and failback. All-member failure holds the current selection and suppresses retries for 15 seconds, then 60 seconds, then five minutes.
- Propagates internal probe context through nested groups so health traffic does not recursively create health sessions.
- Adds policy validation, Leaf config generation, GUI editing and round-trip tests for the optional fallback URL.

## Compatibility boundary

- Ordinary non-stable Leaf failover is unchanged.
- Existing fallback groups need no new field. They use the default health URL only after real traffic reaches a stale group.
- `expected-status` is not exposed; 1xx, 2xx and 3xx are healthy, while 4xx, 5xx and transport/TLS/HTTP errors are unhealthy.
- A failed user request now fails at the selected member instead of trying later members. This prevents destination-specific failures from changing the user's intended route.
